Learning Blog Checkpoint 5

MAR 3, 2021

I took a dataset that was generated from an accounting application called bill.com. I downloaded the data into an excel spreadsheet from the accounting program. The data consisted of transactions between 250 suppliers of food related products from famers, ranchers and fisherman for the year 2020. The transactions were for two manufacturing sites; one in Colorado and the other in Texas. The task was to present the data to a controller, in order to see what suppliers were associated to what manufacturing sights. I wanted to use a pivot table to present the data but without a key identifier for each supplier, it was not possible. So I went about cleaning the data, and assigning a key for each supplier; C for Colorado and T for Texas. I then applied a Vlook up to the reference page which auto populated the keys to the original dataset. I then used a filter to sort out the data, and make it presentable. This little project was my first attempt at cracking a real world problem. I actually started this project as a way to help my wife solve an issue that arised over a weekend. She works in the accounting department for a company out if Denver. She was called by the controller to compile the report above and submit it as soon as possible. When I observed her entering all of the data manually, I thought to myself there must be a faster way to do this. Last semester, I was in Data Wragling, where I was tasked with creating pivot tables in excel. I when about contacting the tutor, who helped me last semester to help with learning this new applicaiton. Unfortunately, there was still quite a bit of manualy data entry that had to be applied, being that a key has to be used in order for a pivot table to properly group different datasets together. Once the referece key was established, the process moved very smoothly. After I was done, I sent the document to my wife for her to keep as a template in future projects.
